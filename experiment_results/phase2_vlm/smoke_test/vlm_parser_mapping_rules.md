# VLM Parser Mapping Rules

To account for conversational responses and token suffixes generated by FastVLM-1.5B, we implement a robust substring-based mapping parser.

## Rule Priority
1. **Lowercase Normalization**: Convert the entire output string to lowercase.
2. **Gesture Phrase Matching**: Match specific multicharacter phrases first (e.g., `"swipe left"` over `"swipe"`).
3. **Keyword-Combination Matching**: If no exact phrase matches, search for combinations of keywords (e.g., both `"swipe"` and `"left"` present anywhere in the output).
4. **Majority Vote Aggregation**: Perform frame-by-frame parsing for 5 frames and take the majority class label. If no class gets a clear majority or all map to `-1`, return `UNKNOWN` (`-1`).

---

## Pattern-to-Label Mapping Rules

### 1. SWIPE_LEFT (Label `0`)
- **Direct Phrases**: `swipe left`, `swiping left`, `swipe_left`, `swiping_left`, `swipe left gesture`
- **Fallback Keyword Combination**: `swipe` AND `left` present in the text.

### 2. SWIPE_RIGHT (Label `1`)
- **Direct Phrases**: `swipe right`, `swiping right`, `swipe_right`, `swiping_right`, `swipe right gesture`
- **Fallback Keyword Combination**: `swipe` AND `right` present in the text.

### 3. ROLL_FWD (Label `2`)
- **Direct Phrases**: `rolling hand forward`, `roll forward`, `roll fwd`, `rolling forward`, `roll_fwd`
- **Common Hallucinated Suffixes**: `rollumont`, `rollplayable`, `rollversed`, `roll`
- **Fallback Keyword**: `roll` present anywhere in the text.

### 4. STOP_SIGN (Label `3`)
- **Direct Phrases**: `stop sign`, `stop hand`, `stop gesture`, `open palm`, `stop signal`, `stop_sign`
- **Common Hallucinated Suffixes**: `stop signing`, `stop drawing`, `stop signifies`, `stop signgesture`
- **Fallback Keyword**: `stop` present anywhere in the text.

### 5. UNKNOWN (Label `-1`)
- If none of the above patterns are matched, the parser defaults to `-1` (UNKNOWN).
